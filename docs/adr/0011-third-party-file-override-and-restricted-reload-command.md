# ADR-0011：三方插件文件覆盖与受限重载命令的安全边界

**状态**：已接受

## 背景

FR-15"三方插件文件覆盖兼容"基于通道B（[ADR-0010](0010-file-tree-hosting-blob-channel.md)）：在"镜像落盘"之上，为无法改源码的三方插件多做两件高风险的事——**覆盖前备份**、**覆盖后执行管理台预设的重载命令**。

后者本质是"对所有子服的远程命令执行面"，对抗式评审判为整个产品的最高风险点。本 ADR 用一组硬约束把它锁成合法且可控；放开其中任一约束即越界（无鉴权 RCE，或滑入 P3 发布编排）。

## 决策

1. **命令执行 gate 在鉴权（[ADR-0009](0009-control-plane-auth-pulled-forward.md)）**：鉴权未落地前，模式二只做"备份 + 整文件覆盖 + 人工/插件自身重载"，**agent 执行下发命令的能力不上线**。
2. **禁 shell、命令受限**：`reloadCommand` 只能是 Bukkit/Bungee **控制台命令**，经 `PlatformAdapter.dispatchConsoleCommand` 派发；agent-core 与适配器**不引入任何进程/shell 执行 API**（禁 `Runtime.exec`/`ProcessBuilder`）——物理上无法落到 OS shell。
3. **agent 本地白名单（反向制衡）**：命令首 token 白名单放 **agent 本地 config**、不由控制面下发——控制面被攻破也无法越白名单提权。**默认空白名单**；单条命令，禁 `; & | > < $` 换行等元字符与管道（防注入）。
4. **路径限死**：覆盖路径用 `Path.normalize().startsWith(Path)`（Path 级，非字符串前缀）校验，限定在 `plugins/<plugin>/` 内；禁 `..`、禁绝对路径/盘符/UNC、禁覆盖 `.jar` 与 server 关键文件（`server.properties` 等）；**Windows 专项**：拒 `:`（盘符/ADS）与保留设备名、规范化后不区分大小写比较。agent 为最终权威，控制面按同规则早校验。
5. **备份/回滚**：覆盖前备份被改/删文件并记"原本是否存在"；回滚**只还原文件 + manifest + 告警，绝不在回滚里重放重载命令**（命令本身可能就是失败根因）。崩溃后向控制面目标态**收敛重做**（幂等），不自动回滚。
6. **不阻塞主线程**：控制台命令在 MC 主线程同步执行会阻塞（很多插件 reload 读盘/连库）。落盘/校验全异步；命令派发**不在主线程同步等结果**——要么只派发不等结果，要么显式接受"重载命令可能造成主线程卡顿"（二选一，实现时写清，不得既宣称合规又主线程同步取结果）。
7. **反馈环防护**：三方插件常在启动/运行时重写自己的 config，会与 agent 覆盖形成"插件写→agent 盖→reload→插件又写"的震荡环。须识别并处理（对受管文件做标记，检测到外部改动时**告警而非盲盖**）。
8. **安全闸**：发布前 **dry-run 只读预览**（将覆盖哪些文件 / 执行什么命令）+ 二次确认。**不做灰度向导**（属 FR-9/P2）；若需"先一台再全网"，用现有 scope 覆盖层手动（先发 server 层、再发 group 层）。
9. **内容校验**：`.js`/`.allin` 等任意文本走**不解析**的保存路径（现有 `validateContent` 跑 `merge.Parse` 只认 yaml/json/properties，会误拒）；md5 计算基准**按字节**定义，防 MySQL TEXT/BOM 往返改字节导致"永远判落后"无限重推。

## 架构边界论证（为何不违"控制面只存事实、禁游戏逻辑"）

控制面存的是"该三方插件的**目标文件态 + 重载方式**"这一**事实**，不编排、不决策、不写游戏逻辑——性质等同于"某配置内容应该是什么"。命令由 **agent 在本地**用**受限白名单控制台命令**派发，是数据面对自身的运维操作，与"agent 收到新配置后自己 reload"同构。白名单 + 禁 shell + 禁 jar 从物理上把可执行范围锁死在"重载类操作"。

**放开任一约束即越界**：碰 jar / 重启进程 → 滑入 P3"版本发布编排"；允许 shell / 宽白名单 / 控制面下发白名单 → 无鉴权 RCE。故上述约束是本特性合法的前提，不是可选项。

## 后果

- `PlatformAdapter` 新增 `dispatchConsoleCommand` 平台能力（壳实现；core 不碰 Bukkit/Bungee/进程 API，守 [ADR-0005](0005-agent-transport-codec-abstraction.md)）。
- 新增备份区与受管文件标记；新增 dry-run 预览端点（随实现同步 [API](../API.md)）。
- 落盘原子写需补 `fsync`（见 [ADR-0010](0010-file-tree-hosting-blob-channel.md) 决策 4）。

## 备选方案

- **允许自由命令 / shell**：无鉴权 RCE。否决。
- **命令支持多条/管道**：扩大注入面。否决；若确需多步重载（如先 unload 再 load），改为"下发命令数组、逐条白名单校验后串行派发"，并单独评估安全代价——留待真有需要时再议。
- **回滚里重放重载命令**：命令本身可能是失败根因，重放会二次伤害且与"不自动回滚"自相矛盾。否决。
- **覆盖 jar / 重启进程**：越界进 P3 发布编排。否决，碰到即停下确认。
