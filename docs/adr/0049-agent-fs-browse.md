# ADR-0049：agent 只读交互式文件浏览（FR-109/110）

**状态**：已接受

## 背景

配置工作台（FR-111）要做成双面板 Xftp 式交互：左受管配置树、右某在线服**实时浏览**真实落盘的 `plugins/`。这需要 agent 提供一种「按需逐层展开、点开才读」的浏览能力——运维点一个目录才列它的子项、点一个文件才读它的内容，像在 Xftp 里浏览远端文件系统那样。

现有的反向抓取（[ADR-0027](0027-reverse-fetch-channel-and-security.md)、[ADR-0037](0037-reverse-fetch-managed-task.md)）是**一次性批量 scan**：admin 触发 → agent 把整棵 `plugins/` 树元信息一次性扫全回传成清单（FR-58）。它服务的是「把整盘配置快照入库」的受管任务，整批、非交互、一次拉全。用它撑双面板右侧的交互浏览不合适：每展开一个目录就重扫整树、大目录拉全表，浪费且不是惰加载的语义。

本特性把「现网某台真实 `plugins/` 文件系统」开放给管理台逐层浏览，安全面与反向抓取同级——读取根、路径校验、只读性、体量必须锁死，放开任一即把现网文件系统变成任意读取的跳板。本 ADR 把「浏览能力」作一个整体决策：既定 agent 侧只读浏览原语（FR-109），也定控制面侧代理口径（FR-110），FR-110 不单独立 ADR。

## 决策

1. **agent 新增只读交互式文件浏览，区别于反向抓取的一次性批量 scan。** agent-core 新增三个**只读**浏览原语：① **懒列目录**——给定相对路径列出其直接子项（目录 / 文件 + 名称 / 大小 / 是否目录 / 是否文本），不递归整树；② **读文件树**——按需展开某子树（仍逐层、有界，非整盘一次拉全）；③ **读单文件内容**——读指定文本文件内容回传。三者是「点开才读」的惰加载，与 FR-58 反向抓取「整树一次性 scan + 整批落库」**语义不同、用途不同**：浏览是交互式、按需、不入库的临时查看；反向抓取是受管任务、整批、入库为托管基线。

2. **读取根限定在该服务器 `plugins/` 目录内。** 浏览的根 = agent 真实 dataFolder 之上的 `plugins/` 根（沿 [ADR-0027](0027-reverse-fetch-channel-and-security.md) 决策3 同一根口径）。所有列目录 / 读文件的相对路径都以此为基准，**绝不允许跳出 `plugins/` 根**去读服务器其它目录、世界数据、系统文件。

3. **强校验 path traversal，拒一切越权路径。** 每次列目录 / 读文件前对路径做规范化校验：`Path.normalize().startsWith(根)`（Path 级，非字符串前缀），解析符号链接后仍须落在根内（**禁符号链接逃逸**）；拒 `..` 回溯、绝对路径、UNC、盘符 `:`、Windows 保留设备名（沿 ADR-0027 决策3 / [ADR-0011](0011-third-party-file-override-and-restricted-reload-command.md) 路径口径，含 Windows 专项）。校验失败即拒该次请求，不读、不回传。

4. **纯只读，绝不开写回旁路。** 浏览只读盘、不写盘——不创建 / 不修改 / 不删除任何文件，不提供任何「经浏览接口改盘」的路径。与反向抓取的「纯只读上传」（ADR-0027 决策8）、只读日志回传（[ADR-0040](0040-agent-readonly-log-tail.md)）同源，守架构不变量 §1「agent 只读暴露、不开改配置旁路」。

5. **async 执行，绝不碰 MC 主线程。** 列目录 / 读文件是阻塞磁盘 IO，全程在 TabooLib async 线程执行（沿 ADR-0027 决策8、架构不变量 §5），主线程零阻塞。

6. **大目录分页 / 惰加载，单文件读有界。** 列目录对大目录**分页**返回（带页标 / 游标，不一次性返回上千子项）；只逐层惰加载、不递归整树；读单文件内容受单文件大小上限约束（复用反向抓取的单文件阈值常量），超限不读全文（避免运行时大文件击穿，与 ADR-0037 同患——大 `*.jsonl` / 落盘指标）。排除 `.jar` 与二进制读取（沿 ADR-0027 决策4，浏览只面向文本配置）。

7. **fail-static。** 浏览能力不可用（agent 不在线 / 命令超时 / 控制面不可达）绝不影响 agent 自身运行与玩家进服——浏览是旁路运维能力，失败即失败、不阻断主流程（架构不变量 §5）。

8. **agent 架构约束一条不破。** HTTP / JSON 仍只在适配器层、core 依赖 `HttpTransport`/`JsonCodec`（[ADR-0005](0005-agent-transport-codec-abstraction.md)）；agent 自管身份、不依赖 CoreLib；agent 是最终权威 + 控制面入库 / 校验前同口径再校验（双保险，沿 ADR-0027 决策6）。**改 agent-core → 双端 jar（bukkit/bungee）重建并真机重部。**

9. **控制面侧（FR-110）：新增 admin 只读端点，复用既有命令通道代理浏览。** 控制面不新增传输、不直连 agent：复用 [ADR-0027](0027-reverse-fetch-channel-and-security.md)/[ADR-0037](0037-reverse-fetch-managed-task.md) 的 SSE 唤醒 + `agent_command` 命令生命周期（FR-104 `pending → fetched → done / failed / expired`）代理 agent 的列目录 / 读文件——admin 触发浏览命令 → SSE 唤醒目标 agent → agent 拉命令、async 读盘、回传 → 控制面把结果代理给前端。浏览命令是 admin 写触发（`full` 角色，`readonly` → 403，扩展 [ADR-0026](0026-runtime-api-keys-and-readonly-role.md)）；触发与结果**入审计**（审计记「谁在何时浏览了哪台的哪个路径」，**敏感文件内容不入审计 detail**，沿 ADR-0027 决策7 / ADR-0040 决策5）。

## 理由

- **交互浏览 ≠ 一次性 scan**：双面板右侧是「点开才看」的惰加载，与反向抓取「整树扫全 + 整批入库」是两种语义。强行复用一次性 scan 撑交互，要么每次展开重扫整树（慢、浪费），要么把整盘清单拉到前端再筛（大目录拉全表、带宽 / 内存不可控）。新增惰加载浏览原语才贴合 Xftp 式交互。
- **根限定 + path traversal 强校验**：把现网真实文件系统开放给管理台，最大风险是被当成任意文件读取的跳板。根死锁在 `plugins/` 内、Path 级规范化 + 禁符号链接逃逸，是这条能力的安全底座——放开根或松校验，现网密钥 / 系统文件就能被无差别读出。
- **只读 + async + 分页**：只读杜绝「浏览接口被用来改盘」；async 守住 MC 主线程不阻塞的红线；分页 / 惰加载 + 单文件有界让大目录、运行时大文件不击穿浏览（ADR-0037 真机暴露过运行时大文件的问题）。
- **控制面复用命令通道不新增传输**：agent 是插件、无 HTTP server，不能被控制面反向直连。复用成熟的 SSE 唤醒 + `agent_command` 生命周期（ADR-0027/0037/0040 已三次沿用），零新增监听面、方向与既有一致（agent 只出站拉 / 传）。
- **浏览作整体决策、FR-110 不单立 ADR**：agent 浏览能力与控制面代理口径是一件事的两端（能力 + 暴露），合在一条 ADR 锁安全面更清晰，避免拆成两条决策互相牵扯。

## 影响

- agent-core 新增只读浏览能力（列目录 / 读子树 / 读单文件，壳实现读盘平台 IO；core 不碰平台 IO，守 ADR-0005）——只读、async、根限定、path traversal 校验、分页 / 有界、排除 jar / 二进制。**改 agent-core → 双端 jar 重建 + 真机重部。**
- 控制面新增浏览命令类型（复用 `agent_command` 生命周期）+ admin 只读端点（代理列目录 / 读文件给前端）+ `full` 角色鉴权 / `readonly` 403 + 触发与结果入审计（detail 不含文件内容）。
- 旧 agent 与新控制面：旧 agent 收不到 / 不识别浏览命令（按未知类型忽略，沿既有命令处理），控制面命令超时清理为 `expired`——能力仅在新 agent 上线（真机须双端升级）。
- 是 FR-111 配置工作台双面板右侧「实时浏览在线服 plugins」的底座；随实现同步 `ARCHITECTURE.md` / `API.md`（新端点 + 命令类型）/ `OPERATIONS.md`。
- **关联**：FR-109（agent 只读交互式文件浏览能力）、FR-110（控制面文件浏览端点）。

## 备选方案

- **复用 FR-58 一次性批量 scan 撑双面板右侧浏览**：反向抓取 scan 是整树一次性扫全 + 受管任务编排，用于「整盘快照入库」。拿它做交互浏览：每展开一个目录就触发整树重扫，或一次把整盘清单拉到前端再筛——非惰加载、大目录拉全表、浪费带宽与内存，且语义错位（浏览是临时查看不入库，scan 是入库基线）。被否（交互浏览需独立惰加载原语）。
- **暴露通用文件系统读（不限 `plugins/` 根 / 弱路径校验）**：让 agent 能读它所在机器的任意路径更「灵活」，但这是把现网文件系统变成任意文件读取跳板——一旦端点被绕或控制面被攻破，现网密钥 / 系统文件无差别外泄。违背只读浏览的根限定与最小权限原则。被否（根死锁 `plugins/` 内 + Path 级强校验，能力上限就是「该服 plugins 下的文本配置」）。
- **给 agent 开只读文件浏览 HTTP 端口、控制面直连拉**：控制面直接 GET agent 更直观，但 agent 是插件无 server，开端口 = 新监听面 + 新鉴权 + 新攻击面，与「agent 只出站」网络模型相悖（同 ADR-0040 被否口径）。被否（复用命令通道，零新增监听）。
- **FR-110 单独立一条 ADR**：浏览能力（agent 侧）与代理口径（控制面侧）是同一安全面的两端，拆成两条 ADR 会让根限定 / 鉴权 / 审计等约束分散、互相牵扯。被否（合为一条整体决策）。
