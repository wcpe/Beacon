# ADR-0050：配置中心双面板 Xftp 工作台（FR-111/112/113）

**状态**：已接受

## 背景

配置中心此前是「单面板受管视图」：左侧只看 Beacon 库里**已纳管**的配置 / 文件树，看不到目标在线服 `plugins/` 下**真实磁盘**的全貌。运维想知道「这台服现在盘上到底有什么、哪些 Beacon 管着、哪些没管、磁盘值与期望合并值差多少」，得在「反向抓取」「拓印审核」「文件树预览」三个**分散页**之间来回跳，心智割裂、且每页只解决一块。

[FR-114](../PRD.md) 已先行落地了**纯前端原型 + mock**（`/admin/v1/workbench/*` 假端点），锁定了双面板工作台的 UX（左右拖拽抓取 / 下发、生效预览并排 diff、底部同步队列、覆盖可见化、撤回回滚等）。原型只验交互、不接真链路。

真链路的两块底座也已就绪或在途：[ADR-0049](0049-agent-fs-browse.md)（agent 只读交互式文件浏览，FR-109）+ FR-110（控制面文件浏览端点，复用 [ADR-0027](0027-reverse-fetch-channel-and-security.md)/[ADR-0037](0037-reverse-fetch-managed-task.md) 命令通道 + FR-104 命令生命周期）让右面板能**实时浏览**某在线服真实 plugins。本 ADR 定的是把原型升级为**真链路**的工作台架构，核心是回答一个问题：**前端接什么后端？新造聚合 BFF，还是贴已有分散端点？**

## 决策

1. **配置中心改双面板工作台。** 左 = 受管配置树（Beacon 库里已纳管的配置 / 文件树）；右 = 某在线服 `plugins/` 的**实时浏览**（真实磁盘，经 FR-110 控制面浏览端点 → FR-109 agent 只读浏览，见 [ADR-0049](0049-agent-fs-browse.md)，懒列目录 / 读文件、async、限 plugins 根、拒 `../`）。两面板对照，一屏看清「管了什么 vs 盘上有什么」。

2. **API 形态（本 ADR 的核心决策）：前端改接既有分散端点，不新造 `/admin/v1/workbench/*` 聚合 BFF。** 工作台的每个能力都映射到一条**已存在**的端点 / 工作流，前端在客户端做编排与拼装：

   | 工作台能力 | 接的既有后端 | 依据 |
   |---|---|---|
   | 左面板受管树 / 编辑 | 既有配置 CRUD（config_item / 文件树） | FR-1 / FR-23 |
   | 右面板实时浏览真实 plugins | 控制面文件浏览端点 → agent 只读浏览 | FR-110 / FR-109、[ADR-0049](0049-agent-fs-browse.md) |
   | 生效预览（逐键来源 provenance） | `GET /admin/v1/configs/effective` | FR-22、[ADR-0013](0013-admin-effective-config-preview-and-provenance.md) |
   | 右→左拖（抓取入库） | 反向抓取受管任务（scan/submit 两段式） | FR-58~60、[ADR-0027](0027-reverse-fetch-channel-and-security.md)/[ADR-0037](0037-reverse-fetch-managed-task.md) |
   | 左→右拖（下发） | 入库为该层覆盖 → 正常下发，受**拓印单人自审门**门控 | FR-46、[ADR-0027](0027-reverse-fetch-channel-and-security.md) |
   | 底部同步队列 | `agent_command` 命令生命周期 | FR-104 |
   | 差异 / 未纳管文件标色 | 有效树比对 + 全量预览追踪态 | FR-45 / FR-68 |

   **原型期 `/admin/v1/workbench/*` mock 仅前端 dev 用，接真后端时退场**——不固化为真实聚合层。

3. **FR-112：配置文件真详情多标签编辑器。** 双击文件进真详情子路由 `/configs/:id`（现 `/configs/:id` 的重定向改真路由），聚焦编辑器，复用既有配置内容 / revisions / 保存确认（[FR-67](../PRD.md) 单文件在页编辑轻量确认）+ Monaco diff / 历史，保留多标签横切 + 局部面包屑 / 返回。

4. **FR-113：三页合一 IA。** 删 `/reverse-fetch` `/imprint` 两条路由 + 侧栏 2 叶子，反抓 / 拓印功能**并入双面板工作台**；`navModel`（导航单一真源）与命令面板（CommandPalette）随之收敛；旧链 404 → 重定向配置中心。**退役已孤立的 `ConfigsPage`**（旧单面板被工作台取代后无引用）。**反抓 / 拓印的后端复用不变**（FR-58~60 / FR-46 端点与契约一字不动），只是前端入口从独立页迁进工作台。

5. **撤回 / 回滚 / 操作日志的后端不在本 ADR。** 工作台原型（FR-114）展示的「逐条 / 批量撤回 + 文件回滚 + 操作日志」真实后端能力（可逆操作快照 + 撤回端点 + 幂等 / 并发 / 事务边界）由 **[ADR-0051](0051-config-operation-undo.md)**（配置操作级撤回子系统，FR-116）定义，本 ADR 只引用、不展开。

## 理由

- **贴已有端点 = 不漂移「复用」原则、最省维护。** 工作台要的每块能力（合并预览 / 抓取 / 下发 / 命令生命周期 / 比对）后端**已经有权威实现**。前端按需调这些端点、在客户端拼装，是薄编排；新造 `/workbench/*` 聚合 BFF 则要在控制面**重复编排**已有 service、多一层聚合契约要维护、还得新立一个「工作台视图」真源去和既有真源对齐——纯属在已有事实上叠一层、违背简单优先与范围纪律。
- **mock 不固化为真后端。** 原型 mock 是为先锁 UX、喂前端测试而存在（FR-114/FR-115），把它升格成真实聚合端点等于把临时脚手架永久化，必腐蚀代码库。接真后端即退场。
- **三页合一收敛心智。** 反抓 / 拓印 / 文件树预览本就是「围绕某服真实 plugins 做对照与同步」的三个切面，分散成三页是历史包袱；并入双面板后「看（右面板浏览）→ 比（差异标色 + 生效预览）→ 抓 / 下发（拖拽）→ 追踪（底部队列）」在一处闭环，后端一行不改。
- **不破坏后端契约。** FR-112/113 是前端 IA + 路由调整 + 客户端编排；除 FR-109/110 的浏览链路（独立 ADR-0049）外，配置 / 反抓 / 拓印 / 命令 / 比对的后端端点全部沿用。

## 架构边界论证（为何不违「控制面只存事实、禁游戏逻辑」）

工作台是前端对**既有只读查询 + 既有受管写工作流**的重新组织：右面板浏览是数据面对自身 plugins 的只读上报（ADR-0049）；生效预览是 ADR-0013 的只读合并查询；抓取走 ADR-0037 受管任务、下发走 FR-46 单人自审门——都不新增编排、不写游戏逻辑、不开「改配置 / 改 zone」旁路。**不新造聚合 BFF** 进一步避免在控制面引入「工作台」这类业务态聚合层。不触碰灰度 / canary / 流量调度 / 版本编排（仍属 P2/P3，不碰、不预留字段）。

## 影响

- **前端**：配置中心重构为双面板工作台；新增 `/configs/:id` 真详情路由（取代旧重定向）；删 `/reverse-fetch` `/imprint` 路由 + 侧栏 2 叶子；`navModel` / CommandPalette 收敛；旧链重定向配置中心；**退役 `ConfigsPage`**；原型 `/admin/v1/workbench/*` mock 退场，改接既有分散端点。
- **后端**：除 FR-110 浏览端点（依赖 FR-109，见 ADR-0049）外，**无新增 / 无契约变更**——配置 CRUD / `configs/effective` / 反抓任务 / 拓印下发 / `agent_command` / 有效树比对全部复用。
- **关联**：FR-111（双面板工作台真链路）、FR-112（真详情多标签编辑器）、FR-113（三页合一 IA）；撤回真后端见 [ADR-0051](0051-config-operation-undo.md)（FR-116）；右面板浏览见 [ADR-0049](0049-agent-fs-browse.md)（FR-109/110）。

## 备选方案

- **落 `/admin/v1/workbench/*` 真实聚合 BFF 端点**：在控制面建一组工作台专用聚合端点，把左右面板数据 / 抓取 / 下发 / 队列在服务端一次拼好回前端。否决——重复编排已有 service、多一层聚合契约的维护成本、还需为「工作台视图」立新真源去对齐既有真源（配置 / 任务 / 命令各有权威），是在事实上叠业务聚合层；选前端贴已有端点更简单、不漂移 PRD「复用」原则、控制面不增业务态。
- **保留原型 mock 当真后端**：把 FR-114 的 `/admin/v1/workbench/*` mock 直接接成真实端点。否决——mock 是临时脚手架（先锁 UX / 喂测试），固化即永久腐蚀；接真后端必须退场、改贴既有端点。
- **不合并、三页各自接真链路**：反抓 / 拓印 / 文件树预览各自独立页分别对接真后端。否决——心智仍割裂，且与 FR-113「三页合一」目标相悖；双面板本就是把这三切面收敛进一处闭环。
