# ADR-0026：运行时 API 密钥与管理面只读角色（落库只存哈希，扩展 ADR-0009）

**状态**：已接受

## 背景

外部业务管理后端需要用一把**只读**密钥调 `/admin/v1/*` 读取拓扑 / 实例 / zone 等事实，且**不具备任何写权限**。

现有管理面鉴权（[ADR-0009](0009-control-plane-auth-pulled-forward.md)）是**单操作者**模型：账号口令登录换 HMAC 签名令牌、凭据走 env 不落库、无角色、无主动吊销（注销需换密钥 / 重启，见 [docs/specs/admin-auth.md](../specs/admin-auth.md) §6）。它满足"人在管理台操作"，但无法表达"给外部服务签发一把可运行时创建 / 吊销、且受限为只读的长期密钥"。

## 决策

1. **新增凭据类型：运行时 API 密钥，落 MySQL 新表 `api_key`。** 与 ADR-0009 的"操作者口令走 env 不落库"**并存不冲突**——操作者口令是单 bootstrap 凭据、天然属配置；API 密钥要运行时增删、必须持久化，env / 重启模型无法承载。表用 GORM 可移植写法（角色 `VARCHAR` + 应用层校验、可空时间列表达过期 / 最近使用、软删哨兵沿用 [ADR-0008](0008-config-soft-delete-and-effective-md5.md)），无 `ENUM/SET/JSON` 列、无方言专有类型，可切 Postgres。
2. **只存哈希，明文仅一次。** 明文 = `bk_` 前缀 + 256-bit 随机串（`crypto/rand`），库内只存其 **SHA-256** 摘要与一段非机密前缀片段（供列表识别）。明文仅在创建 / 重置时一次性返回，**之后绝不可二次读取**；明文不入库、不入日志、不入审计 detail。选 SHA-256 而非 bcrypt：密钥是全熵随机（非弱口令），无需加盐慢哈希，且定长摘要可建唯一索引做 O(1) 查库认证。
3. **密钥只能重置、不能找回。** 无任何"回读明文"端点；密钥遗失 / 泄露的唯一补救是 `reset`（轮换：换新明文，旧明文随哈希变更立即失效）。
4. **两级角色 full / readonly。** 登录令牌恒为 `full`（等同现操作者）；API 密钥按其落库记录取角色。
5. **只读拒写 = 鉴权中间件统一方法级裁决。** `readonly` 角色访问写方法（POST/PUT/PATCH/DELETE）端点一律 **403 FORBIDDEN**，GET/HEAD/OPTIONS 放行；handler 不散落角色判断。语义区分：缺失 / 错误 / 过期 / 吊销密钥 → **401**（未认证），合法但只读越权写 → **403**（已认证但禁止）。
6. **无状态、不引重型件**（守[架构不变量](../../.claude/rules/architecture-invariants.md) #2 简单优先）：密钥校验 = 查库比对哈希，**真源在库、吊销即时生效**；不引 Redis / 会话存储 / OAuth / 鉴权框架。"最近使用"以库内 `last_used_at` 自身做节流（至多每分钟写一次），不另加进程缓存（本规模属过早优化）。
7. **分层不破：** `internal/apikey` 叶子包做生成 / 哈希纯函数；新表 CRUD 在 `repository`；创建 / 吊销 / 重置的校验 + 审计事务在 `service`；中间件经 `ApiKeyVerifier` 接口依赖 service（依赖倒置），`internal/auth` 仍是仅依赖 stdlib 的叶子包（登录令牌）。

## 理由

- 满足"外部服务只读接入"刚需，且把"无意中给写权限 / 泄密"两类风险挡在中间件与存储层（统一裁决 + 只存哈希 + 明文一次）。
- 与 ADR-0009 正交补充而非推翻：操作者 env 凭据决策不变，只是**新增**一类必须落库的运行时凭据。
- 范围克制：只做"只读 vs 读写"两级 + 密钥增删查 + 创建 / 吊销 / 重置审计，刚好够外部只读接入，不镀金。

## 后果

- [PRD](../PRD.md) 新增 **FR-42**（P2）；[ARCHITECTURE](../ARCHITECTURE.md) §3 新增 `api_key` 表、§4 补 API 密钥鉴权说明、§2 模块图补 `internal/apikey` 叶子包；[API.md](../API.md) 新增 `/admin/v1/api-keys` 端点与密钥鉴权头 / 403 语义。
- ADR-0009 保持"已接受"、**不改正文**；本 ADR 扩展其鉴权模型（单操作者 → +可吊销的只读 / 读写 API 密钥）。`docs/specs/admin-auth.md` §6 中"吊销需重启 / 多账户属后续"的局限，对 API 密钥这一新凭据类型由本 ADR 解除（操作者登录模型仍不变）。
- 审计新增动作 `apikey.create` / `apikey.revoke` / `apikey.reset`、对象类型 `apikey`，复用既有 `audit_log` 与审计页过滤。

## 备选方案

- **复用登录令牌签发只读令牌**：无状态签名令牌无法运行时吊销（除非换密钥使全体失效）。否决。
- **细粒度资源 / 字段级权限、按端点 scope、RBAC 矩阵**：超出"只读 vs 读写"刚需、属镀金；需要时再单独提。否决。
- **bcrypt 哈希**：对 256-bit 全熵随机密钥无安全收益，且每请求慢哈希、无法索引查库。否决。
- **密钥放 env / config**：无法运行时创建 / 吊销 / 重置。否决。
- **自动轮换 / 速率限制 / 多租户**：本批范围外，未来按需新增。否决（当前不留占位）。
