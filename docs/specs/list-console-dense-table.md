# 功能规格：列表页 A+E 密集控制台范式

> 状态：开发中　·　关联 PRD：FR-106　·　分支：feature/fr-106-list-console

## 1. 背景与目标
管理台五个列表页（服务器 / 审计 / 告警事件 / 密钥 / 环境）当前各用「筛选 Card + 表格 Card +（可选）分区 Card」竖摞，卡片外壳层层嵌套、留白多、信息密度低，运维一眼扫不到关键计数与全量数据。本 FR（P2，增强 FR-65/FR-42/FR-53/FR-89，依赖 FR-105 已落地的两层页眉）把这五页统一改为「A+E 密集控制台」范式：去卡片化 + 顶部汇总条 + 内联吸顶工具栏 + 裸密表，提升密度与可扫读性。纯前端，不碰后端/agent。

## 2. 需求（要什么）
- 范围内（五页逐页改造）：
  - 去三层 Card 外壳：取消「筛选 Card」「表格 Card」「分区 Card」，内容直接铺在页内容区。
  - 顶部汇总条 `SummaryStrip`：一排紧凑 metric（`bg-secondary` 无边框小卡，标签 11px、数值 ~20px），指标只用现有前端数据/端点派生，**不加任何新后端**。
  - 内联工具栏：原「Label+控件」筛选压成一行紧凑控件，吸顶（`sticky top-0`），保留全部既有筛选维度与「查询」。
  - 裸密表：复用现有 `DataTable`；服务器页未分配实例整行浅黄（`bg-amber-50`）不回归；服务器版本/agent 合一列（`1.20.1 · 0.15.0`，不一致沿用 FR-86 黄框）；列多时外层 `overflow-x-auto` 横向滚动。
- 不做（范围外）：
  - 不动两层页眉框架（PageHeader / 环境 store）；主操作已在 FR-105 上提页眉，保持。
  - 不加新后端端点、不引新依赖、不新 ADR。
  - 告警事件页保留既有时间线（信息流）形态，不强行改表；仅去卡片 + 加汇总条 + 内联工具栏。
  - 不动 FR-107/108 涉及的其它页。

## 3. 设计（怎么做）
- 新增纯展示公共组件 `web/src/components/SummaryStrip.tsx`：接收 `items: { label, value, tone? }[]`，渲染一排 `bg-secondary` 小卡。`tone` 可选语义色（如 `warning`/`danger`/`success`/`muted`）用现有 CSS 变量 token，不硬编码颜色。
- 各页汇总条派生逻辑：
  - 服务器：总实例 / 在线 / 失联 / 排空 / 未分配（从已拉 instances + drains + offline 列表派生）。
  - 审计：本页条数 / 总数（端点返回 total）。
  - 告警事件：未处理无字段——改用 本页 / 各级别（info/warning/critical）计数（从已拉列表派生）。
  - 密钥：总数 / 启用(active) / 即将过期(active 且 expiresAt 在 7 日内)。
  - 环境：环境数。
- 服务器表版本列合一：抽 `versionAgentCell`，`version · agentVersion`，agent 版本不一致时沿用 FR-86 黄框逻辑。
- 内联工具栏：把原 form 内 `Label` 去掉或压成 `sr-only`/占位，控件并排一行；`form` 加 `sticky top-0 z-10 bg-background`。
- 表格横向滚动：`DataTable` 外层包 `overflow-x-auto`。

## 5. 验收标准
- 五页密度提升：无 Card 外壳、有顶部汇总条 + 内联工具栏 + 裸密表。
- 行为不回归：⋯ 操作菜单（详情/日志/重同步/drain/改派/下线）、5s 轮询、点行 Sheet、各筛选、审计导出 csv/json、密钥/环境新建对话框等照常。
- 服务器未分配整行浅黄不回归；版本/agent 合一列且不一致黄框沿用。
- 汇总条计数正确（派生自现有数据，无新后端）。
- i18n 无缺键、暗色正常；`cd web && pnpm test` + `pnpm build` 绿。
- 真机逐页验：本 worktree 不做，交主控整合后串行真机。
